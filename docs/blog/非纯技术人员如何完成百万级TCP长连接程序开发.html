这是本人接触到的一个不太典型的案例，以此记录告诉大家：完成一个了不起的项目，有时候并不需要有很深厚的技术。

  1. 人物及环境简介

    1. 何君：非纯技术出身，但又会写点代码，没有TCP自定义消息编码解码技术基础。但现在他要完成一个有技术含量的项目A
    2. 谭码农：t-io的作者
    3. 某QQ群：一群闲杂人中夹着一些民间高手

  2. 项目A简介

    简单点说就是扫码枪（TCP客户端）会发消息到项目A（服务端），消息格式：4个字节的消息头 + json格式的消息体。消息头用字符串保存消息体的长度。

  3. 何君如何完成

    1. 拷贝showcase整个工程到自己的新工程
    2. 修改包名以及类名，部分业务处理调整，删除showcase中的业务部分代码
    3. 启动服务器，扫码枪发消息过来，一步步调试。发现解码不对，向谭码农求助，谭码农改了几行代码，解码通过
    4. 在业务代码中向扫码枪发消息。发现编码不对，向谭码农求助，谭码农改了4行代码（其实就是删两行代码再加两行代码），编码通过
    5. 顺利通过
